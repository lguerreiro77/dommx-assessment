
# DOMMx Assessment Completion Manual

## Purpose
This manual explains how to correctly complete a DOMMx assessment using the interface implemented in `renderer_assessment.py`.

It provides:
- step‑by‑step instructions
- interface explanation
- assessment rules
- practical tips
- common mistakes to avoid

The goal is to ensure the assessment reflects the **real governance maturity** of the organization.

---

# 1. Assessment Structure

The DOMMx assessment follows this hierarchy:

**Domain → Question → Action → Procedure**

Each domain represents a governance capability.

Examples:

- Data Governance
- Data Architecture
- Data Security
- Data Operations
- Data Quality

Each domain contains several questions which trigger actions and procedures based on the maturity level selected.

---

# 2. Starting an Assessment

Steps:

1. Register in the system
2. Provide electronic consent
3. Log in
4. Select the project
5. Read supporting documentation
6. Open the Assessment screen
7. Start answering the questions

---

# 3. Interface Layout

The screen contains:

- domain title
- domain description
- question text
- question objective
- response selector
- maturity buttons
- actions triggered by answers
- procedure descriptions
- comment field
- navigation buttons
- save buttons
- submission button

The interface saves answers progressively.

---

# 4. Answering Questions

Steps:

1. Read the question carefully
2. Evaluate current practices
3. Select the maturity level
4. Add comments if needed
5. Consult reference documentation

---

# 5. Maturity Levels

Typical interpretation:

Level 0 – Practice does not exist  
Level 1 – Initial practice  
Level 2 – Defined practice  
Level 3 – Managed practice  
Level 4 – Optimized practice  

Always answer based on current reality.

---

# 6. Evidence

Include evidence when possible:

- policies
- governance committees
- stewardship roles
- quality controls
- monitoring processes

---

# 7. Navigation

Users can move using:

Next question  
Previous question

Answers remain stored during navigation but may be lost if the screen is closed without saving.

---

# 8. Saving

Answers are saved automatically while the user remains on the page.

Mandatory questions must be answered before submission.

---

# 9. Domain Completion

After all questions are answered the system calculates:

- maturity score
- recommendations
- report outputs

---

# 10. Results

The system allows exporting the assessment report after completion.

---

# 11. Tips

✔ answer with evidence  
✔ maintain consistency across domains  
✔ avoid aspirational answers  

---

# 12. Common Mistakes

❌ inconsistent answers  
❌ missing comments  
❌ overestimating maturity  

---

# 13. Philosophy

The DOMMx assessment identifies governance gaps and supports governance improvement roadmaps.
